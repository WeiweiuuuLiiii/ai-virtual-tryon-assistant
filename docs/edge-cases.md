# Edge Cases

## Image Quality

- User uploads blurry full-body photo
- User uploads mirror selfie with poor lighting
- User uploads clothing image with background clutter
- Garment is folded or not fully visible
- Shoes/accessories are partially cropped

## Body Model / Measurement Issues

- User enters unrealistic measurements
- User leaves measurements incomplete
- Full-body photo does not show complete body
- Loose clothing hides body shape
- User’s body proportions are difficult to estimate accurately

## Outfit Composition

- User uploads only one item
- User selects incompatible categories
- User selects too many items
- Outfit is missing shoes or bottom/top layer
- Same item is selected twice

## Context Mismatch

- Outfit looks good visually but does not match weather
- Outfit is too formal or too casual for selected occasion
- Outfit is uncomfortable for travel or walking-heavy days
- Item is stylish but has low repeat-use likelihood

## Recommendation Trust

- AI recommendation is too vague
- AI gives overly confident advice
- User disagrees with recommendation
- Recommendation does not explain tradeoffs clearly
- User needs control over final decision

## Privacy

- User uploads sensitive body image
- User wants to delete uploaded image
- User does not want body data stored
- User wants local-only mode

## Product Principle

When uncertain, the system should say it is uncertain.  
The goal is to support user judgment, not replace it.
